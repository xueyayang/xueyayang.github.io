C/C++中测试代码时间
===================

问题
====
对程序效率注意的时候，自然要测试某个代码块的执行时间。如何计算？


方法
====
这个方法在C/C++中通用，精度可以在ms。一般来说够用了。
{% highlight c %}
#include <time.h>

clock_t start,end;
start = clock();
    ....
 <code snippet>
	....
end = clock();
printf("execution time:%f [ms]",(end -start)*1000/CLOCKS_PER_SEC);
{% endhighlight %}

注意：输出时，千万要`%f`。不然值很奇怪，还不好找哪里出了问题。

总结
====
简单有效方便。以不变应万变。
